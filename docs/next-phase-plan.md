# VH-01 下一阶段计划

## 0. 当前有效主线

本文件中较早关于“读取层 / query-aware / 候选约束执行”的内容保留为历史记录，但当前有效主线已经切换为：

$$
R_{i,j,l}
\rightsquigarrow
\text{局部响应几何}
\rightsquigarrow
\mathcal L_c
\rightsquigarrow
\mu_c.
$$

因此，当前阶段的“下一步”不再是继续打磨离散读取规则，而是继续补全连续测度路线。最新的实施蓝图见 [continuous-measure-plan.md](/home/void0312/AIGC/VH-01/docs/continuous-measure-plan.md)。

## 1. 原则

下一阶段不再以“继续调几个参数看看”为主，也不提前切换到更强基线。

正确顺序应当是：

1. 先把与理论主线直接对应的底座补齐。
2. 再做功能性验证，检查这套结构是否真的在按理论工作。
3. 只有在底座闭环已经成立之后，才允许做少量校准。
4. 只有当底座闭环成立但能力明显受限时，才考虑切换强基线。

因此，当前阶段的关键词不是“优化”，而是“补全”。

## 2. 当前底座还缺什么

虽然我们已经有了：

- `X_{1:T} -> Z_{1:T}` 的表示链路
- `R_{i,j,l}` 与 `E_dyn` 的残差主线
- 条件打分代理与部分支持收缩

但从理论上看，底座仍然还没有完全闭环。当前剩余的关键缺口主要是以下几类。

### 2.1 理论核心映射仍未显式对象化

目前我们已经有了离散读取对象，也已经有了最小读取约束执行链，但仍然缺少更核心的一步：把理论中的

- `\mu_0`
- `\mathfrak T_c`
- `\mu_c`

与工程中的具体对象做更严格的一一映射。

当前更准确的状态是：

1. `condition_candidate_logits` 与 `condition_alignment_energy` 是离散代理，不是显式的连续测度对象；
2. `CandidatePosterior`、`CandidateSet`、`query_responsive_selection` 是读取层对象，不是理论本体；
3. 因此下一阶段不能只围绕读取规则继续展开，而需要回到映射本身。

### 2.2 查询响应读取规则已经出现，但仍不是最终最优

目前我们已经有了正式的查询响应读取规则：

> 观测读取集合 ∩ 查询规划核心，再在其中按观测后验选择

当前实验已经表明：

1. `support_top1` 还不足以直接替代查询条件；
2. 查询响应读取规则比 `support_top1` 更合理，也更贴合当前工程目标；
3. 但它仍未达到最优生成质量。

### 2.3 生成执行闭环已成立，但仍然是轻量版本

我们现在已经形成了最小闭环：

$$
\text{条件打分代理} \to \text{离散读取结果} \to \text{读取约束执行/选择}
$$

但它还只是分析级别的最小执行链，而不是一个完整、稳定、可部署的生成策略。

因此下一阶段要做的，不再是“有没有闭环”，而是：

1. 这条闭环在更大范围上是否仍成立；
2. 当前读取规则如何与训练选择准则形成更合理的一致性；
3. 轻量底座是否已经逼近容量上限。

## 3. 下一阶段的严格顺序

### Stage A: 补齐理论底座

这一阶段不做参数扫，不比种子，不换强基线。

这一阶段里，“读取层补全”已经基本完成，但“理论核心映射”还没有完全完成。当前更准确的拆分应当是：

已经完成：

1. 显式实现离散读取对象。
2. 把读取统计写入训练/分析接口，而不是只在评测脚本里临时计算。
3. 接入“读取约束生成/选择”入口，形成最小执行闭环。

尚未完成：

1. 明确训练链路中哪些对象真正对应 `\mu_0` 的代理。
2. 明确条件 `c` 在代码里究竟通过什么结构近似 `\mathfrak T_c`。
3. 明确现有离散后验哪些可以被解释为 `\mu_c` 的有限读取，哪些只是工程拼接分数。

这一阶段的目标不是涨点，而是让系统结构上完整。到当前为止，读取层部分已经基本达成，但理论核心映射仍需继续收束。

### Stage B: 做功能性实验

在底座补齐后，做三类实验：

1. 条件扰动实验  
   检查 `c -> c'` 时，离散读取结果是否发生有结构的迁移。

2. 读取保真实验  
   检查真实轨迹是否稳定落在读取结果内，以及读取结果是否出现合理收缩。

3. 读取约束生成对照  
   比较：
   - 不使用读取约束的生成/选择
   - 使用读取约束后的生成/选择

如果这三类实验没有给出清楚结果，就说明理论底座还没有真正站住。

其中前两项已经开始具备结果：

1. 条件扰动实验已完成第一轮；
2. 读取保真实验已通过支持指标与 constrained rollout 做了最小验证；
3. 读取约束生成对照也已完成第一轮；
4. 读取规则研究也已完成第一轮；
5. 查询响应读取规则也已正式接入执行链；
6. 当前已经可以转入下一阶段的扩展验证。

### Stage C: 下一阶段的少量校准与扩展验证

当前 Stage A 的读取层部分和 Stage B 已经基本成立，因此现在可以进入下一阶段。

这里的“校准与扩展验证”不是无目的扫参，而是只针对主线明确相关的问题，例如：

- 新的 `query_balanced` 默认 checkpoint 在更多查询扰动下是否仍成立
- 训练期 query-aware 观测是否需要从 `true query` 扩展到 `near/far query`
- 默认执行优先 checkpoint 在更大范围上是否仍成立
- 查询响应读取规则与训练目标之间是否需要进一步耦合
- 更强数据/更强生成器上的扩展验证

而且必须满足两个约束：

1. 每次只改极少数参数；
2. 每次改动都要服务于一个明确问题，而不是“看看会不会更高”。

### Stage D: 更强基线或更强生成器

只有在以下条件之一成立时，才考虑切换强基线：

1. 理论底座映射已经补齐，但效果仍明显不足；
2. 已经证明当前轻量底座容量不足，无法承载理论结构；
3. 读取闭环已经成立，需要在更强生成器上验证可扩展性。

现在它不再是“绝对不能碰”的选项，而是下一阶段中的条件性选项。

## 4. 当前阶段禁止事项

为避免重新掉回无意义的试验循环，当前阶段明确不做：

1. 无目的超参数扫描；
2. 单纯为了涨几个点的微调实验；
3. 单纯为了看波动而换种子重复；
4. 在理论核心映射未补齐前直接切换到强基线。

这些动作在当前阶段都会稀释主线。

## 5. 下一阶段立即执行项

基于上述顺序，接下来的直接动作应为：

1. 巩固 `trajectory_point / trajectory_summary_context / response signature` 的对象分工，不再让 `trajectory_state` 混用局部基点与摘要语义；
2. 沿着已经验证有效的 chart-target 路线继续推进，因为 synthetic 与 real bootstrap 都表明：坐标一致性比单纯 trace 修补更关键；
3. 扩展弱形式测试函数与局部几何诊断，确认 `\mathcal L_c^* \mu_c = 0` 的近似恢复不只依赖低阶矩；
4. 在“耦合 synthetic 非对角诊断”已经完成后，推进 `A_c` 的结构建模升级，解决“目标非对角已存在但预测非对角仍偏弱”的问题（`state_cov_proj_dim=8` 与 `response_signature` 上下文首轮尝试均未形成有效突破）；
5. 只有在连续测度链条站稳后，才重新考虑读取、离散化和更强生成器。

因此，下一步的任务不再是“继续 query-aware 扩展验证”，而是：

> 在连续 `\mathcal M_T` 与局部响应到测度恢复这条主线上，把二阶结构、尺度校准与弱形式恢复补齐，再讨论后续离散化。
