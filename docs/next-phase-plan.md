# VH-01 下一阶段计划

## 1. 原则

下一阶段不再以“继续调几个参数看看”为主，也不提前切换到更强基线。

正确顺序应当是：

1. 先把与理论主线直接对应的底座补齐。
2. 再做功能性验证，检查这套结构是否真的在按理论工作。
3. 只有在底座闭环已经成立之后，才允许做少量校准。
4. 只有当底座闭环成立但能力明显受限时，才考虑切换强基线。

因此，当前阶段的关键词不是“优化”，而是“补全”。

## 1.1 当前进度

已完成：

1. 候选集算子已成为正式模块。
2. 最小候选集约束 rollout 执行链已跑通。
3. 第一轮条件扰动实验已完成。
4. 第一轮“候选集约束生成 vs 非约束生成”对照已完成。
5. 第一轮候选集内部选择规则研究已完成。

当前尚未完成：

1. 面向查询响应的候选集内部选择规则设计。
2. 观测后验候选集与规划执行候选集的统一化表达。
3. 面向主线文档的理论-实验闭环总结。

## 2. 当前底座还缺什么

虽然我们已经有了：

- `X_{1:T} -> Z_{1:T}` 的表示链路
- `R_{i,j,l}` 与 `E_dyn` 的残差主线
- 条件支持打分与部分支持收缩

但从理论上看，底座仍然还没有完全闭环。当前剩余的关键缺口主要是以下几类。

### 2.1 候选集内部选择规则还不明确

目前我们已经有了正式的候选集算子 `\mathcal T_\alpha(c)`，也已经有了最小候选集约束执行链。

但问题已经从“有没有候选集”转成了：

> 在候选集内部，究竟应该如何选择最终执行轨迹？

当前实验已经表明：

1. `support_top1` 还不足以直接替代查询条件；
2. 候选集本身作为受限搜索空间是有价值的；
3. 第一轮联合规则研究虽有进展，但最终选择规则仍未固定。

### 2.2 生成执行闭环仍然是最小版本

我们现在已经形成了最小闭环：

$$
c \to \mathcal T_\alpha(c) \to \text{候选受限生成/选择}
$$

但它还只是分析级别的最小执行链，而不是一个完整、稳定、可部署的生成策略。

因此还需要明确：

1. 最终选择规则；
2. 阈值与预算的关系；
3. 候选集约束是否真的优于不约束的正式执行方案。

### 2.3 理论闭环总结还没有整理成统一结论

我们已经有了三类功能性结果：

1. 候选集保真实验；
2. 条件扰动实验；
3. 候选集约束生成对照。

但它们还没有被整理成一份统一的“理论对象 -> 工程对象 -> 实验结论”的闭环总结。

这一点如果不补，就很容易再次回到“只是在跑一些分析脚本”的状态。

## 3. 下一阶段的严格顺序

### Stage A: 补齐理论底座

这一阶段不做参数扫，不比种子，不换强基线。

这一阶段的主要任务已经基本完成：

1. 显式实现候选集算子 `\mathcal T_\alpha(c)`。
2. 把候选集统计写入训练/分析接口，而不是只在评测脚本里临时计算。
3. 接入“候选集约束生成/选择”入口，形成最小执行闭环。

这一阶段的目标不是涨点，而是让系统结构上完整。到当前为止，这一目标已经基本达成。

### Stage B: 做功能性实验

在底座补齐后，做三类实验：

1. 条件扰动实验  
   检查 `c -> c'` 时，`\mathcal T_\alpha(c)` 是否发生有结构的迁移。

2. 候选集保真实验  
   检查真实轨迹是否稳定落在候选集内，以及候选集是否出现合理收缩。

3. 候选集约束生成对照  
   比较：
   - 不使用候选集的生成/选择
   - 使用候选集约束后的生成/选择

如果这三类实验没有给出清楚结果，就说明理论底座还没有真正站住。

其中前两项已经开始具备结果：

1. 条件扰动实验已完成第一轮；
2. 候选集保真实验已通过支持指标与 constrained rollout 做了最小验证；
3. 候选集约束生成对照也已完成第一轮；
4. 候选集内部选择规则研究也已完成第一轮；
5. 当前剩余的核心任务转为：面向查询响应的选择规则设计与闭环总结。

### Stage C: 少量校准

只有当 Stage A 和 Stage B 已经成立后，才允许做少量校准。

这里的“校准”不是无目的扫参，而是只针对理论上直接相关的量做有限调整，例如：

- 候选阈值 `alpha`
- 支持收缩权重
- 候选集选择准则

而且必须满足两个约束：

1. 每次只改极少数参数；
2. 每次改动都要服务于一个明确问题，而不是“看看会不会更高”。

### Stage D: 是否切换强基线

只有在以下条件之一成立时，才考虑切换强基线：

1. 理论底座已经补齐，但效果仍明显不足；
2. 已经证明当前轻量底座容量不足，无法承载理论结构；
3. 候选集闭环已经成立，需要在更强生成器上验证可扩展性。

也就是说，强基线不是下一步，而是更后面的阶段性选项。

## 4. 当前阶段禁止事项

为避免重新掉回无意义的试验循环，当前阶段明确不做：

1. 无目的超参数扫描；
2. 单纯为了涨几个点的微调实验；
3. 单纯为了看波动而换种子重复；
4. 在理论底座未补齐前直接切换到强基线。

这些动作在当前阶段都会稀释主线。

## 5. 立即执行项

基于上述顺序，接下来的直接动作应为：

1. 设计不再明显偏向观测端的查询响应选择规则；
2. 明确观测后验候选集与规划执行候选集的统一方式；
3. 把现有分析结果整理成统一的理论-实验闭环总结；
4. 在此基础上，再决定是否需要极少量、问题导向的校准。

因此，下一步的任务不再是“把底座补出来”，而是：

> 把已经补出来的底座整理成一个可解释、可执行、可总结的闭环系统。
